Vue组件之间的通信方式

Props 父组件通过props向子组件传递数据 子组件通过$emit和父组件通信 props只能是父组件向子组件进行传值 props使父子组件之间形成一个单向的下行绑定 子组件的数据随着父组件的更新而响应式更新

props可以显示定义一个或一个以上的数据 对于接收的数据 可以是各种数据类型 也可以传递一个函数

props属性名规则 若在props中使用驼峰形式 模板中标签需要使用短横线的形式书写

l $emit v-on 子组件向父组件传递数据 $emit 绑定一个自定义事件 事件被执行时会将参数传递给父组件 父组件通过v-on监听并接收参数

l ref/refs 也是实现父子组件通信的 ref用在子组件上 指向子组件的实例 用来通过实例访问组件的数据和方法

l provide/inject 依赖注入 该方法用于父子组件之间的通信 也可以是祖孙组件 层数很深的情况下可以使用这种方式进行传值 不用一层一层的传递数据

Provide和inject是vue提供的两个钩子 和data methods是同级的 并且provide书写形式和data一样 provide用来发送数据或方法 inject用来接收数据或方法

用法： 父组件中 provide(){return{参数:参数}} 子组件中 inject:[‘参数名’]

注意 依赖注入所提供的属性是非响应式的

l $parent/$children

Ø 使用$parent可以让组件访问父组件的实例 (访问的是上一级父组件的属性和方法) 使用$children可以让组件访问子组件的实例 $children并不能保证顺序 并且访问数据不是响应式的

Ø 用法：

n 子组件中在computed定义自定义方法 return this.$parent.参数名 直接使用message

n 父组件绑定事件this.$children[0].message = 值 不需要发送 直接可以调用父组件data内数据

Ø 注意：

n 通过$parent访问到的是上一级父组件的实例 可以使用$root来访问根组件的实例

n 在组件中使用$children拿到的是所有子组件的实例 是一个数组 并且是无序的

n 在根组件#app上拿$parent得到的是new Vue()的实例 在实例上再拿$parent得到的是undefined 在最底层的子组件拿$children是个空数组

n $children的值是数组 $parent是对象

(1) 兄弟组件通信

l $emit/$on eventBus 中央事件总线 适用于父子组件 非父子组件等之间的通信

使用步骤：1创建时间中心管理组件之间的通信 export const EventBus = new Vue()

2 发送事件 export default{components:{事件,事件}}

在组件中发送事件 EventBus.$emit(‘名称’,{参数：参数})

3 在组件中接收事件 EventBus.$on(‘名称’,params=>{参数=参数})

(2) 任意组件通信

l Vuex

Ø 是一个专为vue应用程序开发的状态管理模式库 采用集中式存储管理应用的所有组件状态 并以相应的规则保证状态以可预测方式发生变化

l $attrs/$listeners

Ø A组件是B组件的父组件 B组件是C组件的父组件 如果想组件A给C组件传递数据 隔代传数据的情况该使用什么情况？

n 如果用props/$emit来一级一级传递 可以完成 但是比较复杂 如果使用事件总线 在多人开发或者数据较大 维护很麻烦 如果使用vuex 仅仅传递数据比较浪费

n 针对上述情况 vue引入了$attrs/$listeners 实现组件之间跨代通信

n $attrs继承所有父组件属性(除了props传递的属性 class和style) 一般用在子组件的子元素上

n $listeners:该属性是一个对象 里面包含了作用在这个组件上的所有监听器 可以配合v-on=’$listeners’将所有的事件监听器指向这个组件的某个特定子元素

n inheritAttrs 默认值为true 继承所有父组件属性除props之外的所有属性

n 只继承class属性

Ø 用法：

n A组件 监听事件 在B组件和C组件中直接触发即可

n B组件 使用props获取A组件内的自定义时间或取出数据 使用$emit触发A组件的方法

C组件 使用props获取A组件内的自定义时间或取出数据 使用$emit触发A组件的方法
