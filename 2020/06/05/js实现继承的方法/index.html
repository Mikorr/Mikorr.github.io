js实现继承的方法

1.原型链继承

​ 该方法与属性集中可复用的部分迁移到原型链中 而将不可复用的部分设置为对象自身的属性

2.仅从原型继承

​ 不存在原型链 所有的对象共享一个原型对象

​ 优点：
​ 1.由于该模式在构建继承关系时不需要新建对象实例 效率上会有较好的表现

​ 2.原型链上的查询也会比较快 因为这里根本不存在链

​ 缺点：

​ 对子对象的修改会影响父对象

3.临时构造器

​ 只继承父对象的原型属性 而对于其自身属性则不予继承

​ 可以通过uber属性便捷访问父对象

7.原型继承

​ 直接在对象之间构建继承关系

​ 发挥原型的固有优势

8.扩展与增强模式

9.多重继承

​ 一种混合插入式的继承实现 会按照父对象的出现顺序依次对它们执行属性全复制

10.寄生式继承

  • 该方法通过一个类似于构造器的函数来创建对象

11.构造器借用

  • 可以只继承父对象的自身属性
  • 可以与方法1结合使用，以便从原型中继承相关内容
  • 便于子对象在继承某个对象的具体属性（并且还有可能是引用类属性）时，选择最简单的处理方式

如何选用继承方法？

​ 取决于我们的设计风格、性能需求、具体项目任务及团队。

  • 习惯于从类的角度来解决问题？那么基于构造器的工作模式更合适
  • 只关心该“类”的某些具体实例，则使用基于对象的工作模式更合适。
